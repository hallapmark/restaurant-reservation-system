# AI panuse lühilog

## reservationApi.ts

- AI aitas koostada teenusekihi algse struktuuri ja `fetch`-põhised API väljakutsed.
- Mina kasutasin seda hetkel baaslahendusena ning kontrollisin, et see sobituks projekti frontendi andmemudeliga.
- Hiljem lisasin AI abil ka eraldi päringu lähedaste vabade kellaaegade leidmiseks (`/availability/slots`), et time-discovery jääks samale demo voole toetavaks lisavaateks.
- Broneerimise lisamisel täiendasin AI abil sama teenusekihti ka reservation API väljakutsetega ning veateadete lugemisega backendi ühtsest veavormingust.
